## 2. Page Limits

**CRITICAL AAAI-SPECIFIC DIFFERENCE:** Unlike NeurIPS and ICML, AAAI has historically counted references toward the page limit. However, starting with AAAI-25 and continuing in AAAI-26, **references are on additional pages** and do NOT count toward the 7-page limit.

| Component | Limit |
|-----------|-------|
| Technical content | **7 pages maximum** |
| References | **Additional pages** (unlimited, do not count toward 7-page limit) |
| Reproducibility checklist | **Additional pages** (does not count toward page limit) |
| Acknowledgements | **Omit from review submission**; include only in camera-ready |

From the AAAI-26 submission instructions:
> "Submissions may consist of up to 7 pages of technical content plus additional pages solely for references and the reproducibility checklist; acknowledgements should be omitted from papers submitted for review."

There is no separate "short paper" category in the main track. Student abstracts and posters have different limits (check their specific CFPs).

## 3. File Requirements

### Submission Phase (for review)
- **Format:** PDF only (no source files required at submission time)
- **Paper size:** US Letter (8.5" x 11") -- not A4
- **Resolution:** High-resolution PDF
- **Fonts:** Type 1 or TrueType only; **no Type 3 fonts** (even in figures)
- **Compilation:** Must compile in PDFLaTeX
- **No embedded links or bookmarks** (hyperref and navigator packages are forbidden)
- **No password protection**
- **Metadata:** Clean PDF metadata to prevent de-anonymization; use `\pdfinfo{/TemplateVersion (2026.1)}`

### Camera-Ready Phase (for publication)
- **Source files:** Single `.tex` file (no `\input` command allowed) + `.bib` file(s)
- **Archive:** Single `.zip` file, **maximum 10 MB**
- **Naming:** Source file named with last (family) name of first author
- **Required files:** PDF, `.tex` source, `.bib`, graphics files, `.aux`, `.bbl`
- **Compilation:** Must compile on standard LaTeX 2020 TeXLive
- **Figure formats:** `.jpg`, `.png`, or `.pdf` only (no `.gif`, `.ps`, `.eps`)
- **All fonts must be embedded** in the PDF, including in figures

### Forbidden in Source Files
- Do not include unused files
- Do not include the instructions file or style files
- No intermediate build files

- AAAI-26 Author Kit (AuthorKit26, downloaded from [https://aaai.org/authorkit26-1/](https://aaai.org/authorkit26-1/))

## 4. Desk-Rejection Triggers

Papers will be desk-rejected (rejected without review) for any of the following:

### Content Violations
- [ ] **Placeholder titles/abstracts** -- Submissions with "test", "xyz", or empty titles/abstracts will be deleted; authors will not be allowed to submit a full paper
- [ ] **Substantially changed abstract** -- If the abstract changes qualitatively between the abstract deadline (July 25) and paper deadline (August 1), the paper will be rejected without review
- [ ] **Exceeding submission cap** -- Each author is limited to **10 submissions** across all technical tracks (Main + AI Alignment + AI for Social Impact)
- [ ] **Non-anonymized submission** -- Author names, affiliations, or identifying information present
- [ ] **Over-length papers** -- Exceeding 7 pages of technical content
- [ ] **Missing PDF** at submission time

### Formatting Violations
- [ ] **Wrong paper size** -- Must be US Letter, not A4
- [ ] **Modified style file** -- Any modifications to `aaai2026.sty` or use of commands/packages that alter layout
- [ ] **Forbidden packages used** (see Section 6)
- [ ] **Type 3 fonts** present (even in figures)
- [ ] **Non-Roman alphabet fonts** in text (must be restricted to bitmapped figures)
- [ ] **Overfull boxes** -- Text or equations intruding into margins
- [ ] **Embedded links/bookmarks** (hyperref, navigator)
- [ ] **Computer Modern font** used for body text (must use Times or Nimbus)

### Policy Violations
- [ ] **Dual submission** -- Paper under review or published at another archival venue
- [ ] **Author changes after deadline** -- No authors can be added after August 1
- [ ] **Contact author changes after abstract deadline** -- Cannot change after July 25

## 5. Anonymization Rules

AAAI-26 uses **double-blind review**. The requirements are strict:

### Required Actions
1. **Remove all author names and affiliations** from the submission. Write "Anonymous Submission" as the sole author.
2. **Use the `[submission]` option** in the style package: `\usepackage[submission]{aaai2026}`
3. **Clean PDF metadata** using a metadata-cleaning tool before submitting
4. **Anonymize self-references** -- References to the authors' own previous work must be anonymized whenever the reader could infer authorship
5. **Remove AAAI copyright notice** from footer (the `[submission]` option handles this automatically)
6. **Remove acknowledgements** from the review submission

### Supplementary Material
- Take particular care to ensure supplementary material (especially videos) avoids:
  - Images of the authors
  - Identifiable voices
  - University or lab logos
  - Recognizable campus scenes
- Do not include web pointers in supplementary material (violates both blind review and the no-modification-after-submission policy)

### What Is Allowed
- Citing your own prior work in the third person is acceptable when the citation is necessary
- ArXiv preprints are allowed (see Section 11), though this may reveal identity

```latex
\usepackage[hyphens]{url}                % DO NOT CHANGE
\usepackage{graphicx}                    % DO NOT CHANGE

\usepackage{natbib}                      % DO NOT CHANGE, NO OPTIONS
\usepackage{caption}                     % DO NOT CHANGE, NO OPTIONS

```

### Forbidden Packages
The following packages will cause errors or rejection:

| Package | Reason |
|---------|--------|
| `authblk` | Conflicts with AAAI author formatting |
| `balance`, `flushend` | Alters column layout |
| `CJK` | Non-Roman font issues |
| `float` | Alters float placement |
| `fontenc` | Font encoding conflicts |
| `fullpage`, `geometry` | Alters page layout |
| `grffile` | File handling conflicts |
| `hyperref`, `navigator` | Embeds links (forbidden) |
| `savetrees` | Alters spacing |
| `setspace` | Alters line spacing |
| `stfloats` | Alters float behavior |
| `tabu` | Table formatting conflicts |
| `titlesec` | Alters section formatting |
| `tocbibind` | Table of contents conflicts |
| `ulem` | Text formatting conflicts |
| `wrapfig` | Alters float behavior |
| `bbm` | Introduces Type 3 fonts |
| `multicol` | Column conflicts |
| `color` (if used in text) | Color in text is forbidden |

### Forbidden Commands
`\addtolength`, `\baselinestretch`, `\clearpage`, `\newpage`, `\pagebreak`, `\columnsep`, `\pagestyle`, `\tiny`, `\vspace{-...}`, `\vskip{-...}` (negative values near captions/sections), `\input` (for including paper sections), `\clip`, `\trim`, `\nocopyright`

### Layout Specifications
- **Two-column format** (mandatory)
- **Margins:** Top 0.75", Left 0.75", Right 0.75", Bottom 1.25"
- **Column width:** 3.3" each with 0.375" gutter
- **Body text:** 10pt Times Roman with 12pt leading
- **Title:** 16pt bold, centered, Title Case capitalization
- **Section numbers:** Optional (set `\setcounter{secnumdepth}{0}` for unnumbered, `1` for sections, `2` for subsections)

## 8. Paper Checklist / Ethics Requirements

### Reproducibility Checklist (Mandatory)
All authors must complete the AAAI-26 **Reproducibility Checklist** and include it in the PDF **after the references**. It does NOT count toward the page limit. The checklist covers four sections:

**1. General Paper Structure**
- Pseudocode/outline of AI methods
- Clear separation of opinions vs. facts
- Pedagogical references for background

**2. Theoretical Contributions**
- Assumptions stated formally
- Novel claims stated as theorems
- Proofs included (or proof sketches)
- Empirical demonstration of theoretical claims

**3. Dataset Usage**
- Motivation for dataset choice
- Novel datasets in appendix + publicly available
- Proper citations for existing datasets

**4. Computational Experiments**
- Hyperparameter ranges documented
- Code included in appendix (or will be released)
- Random seed methodology described
- Computing infrastructure specified
- Evaluation metrics formally described
- Number of runs stated
- Variation/confidence measures reported
- Statistical significance tests used

The checklist is provided as `ReproducibilityChecklist.tex` in the AAAI-26 Author Kit and can be `\input` into the main document.

### Ethics Policy
- All authors must honor the [AAAI Publications Ethics and Malpractice Statement](https://aaai.org/conference/aaai/aaai-26/policies-for-aaai-26-authors/)
- All authors must follow the [AAAI Code of Professional Conduct](https://aaai.org/)
- A paper may be rejected if an author violates either policy
- Authors may include an optional **Ethical Statement** section (unnumbered, after appendices, before acknowledgements)

### Paper Structure Order
1. Main content sections
2. Appendices (optional, lettered A, B, C...)
3. Ethical Statement (optional, unnumbered)
4. Acknowledgements (optional, unnumbered; omit from review submission)
5. References (unnumbered)
6. Reproducibility Checklist

## 9. LLM/Agent Usage Policy

AAAI-26 has a specific and strict policy on AI/LLM use:

### Prohibited
- **LLM-generated text in the paper** is prohibited unless the generated text is presented as part of the paper's **experimental analysis** (e.g., studying LLM outputs)
- **AI systems cannot be listed as authors** -- AAAI policy states that "any AI system, including Generative Models such as ChatGPT, BARD, or DALL-E, do not satisfy the criteria for authorship"
- **AI systems cannot be cited as sources** in AAAI papers

### Permitted
- Using LLMs for **editing or polishing** author-written text is allowed
- Authors may "judiciously use generative AI tools" in **preparing** manuscripts, but remain **fully responsible** for all submitted material

### Enforcement
- Any allegation of plagiarism (including LLM-related) will be "thoroughly investigated"
- Sanctions include: rejection/retraction, notification to authors' institutions/employers, and denial of service/access to all AAAI-sponsored meetings
- Authors assume full responsibility for content, including checking for plagiarism and veracity

### New for AAAI-26: AI-Assisted Review Pilot
AAAI-26 introduced an **AI-Assisted Peer Review pilot program**:
- Phase 1 reviews include 2 human reviews + 1 AI-generated supplementary review
- AI reviews contain **no ratings or recommendations**
- AI reviews are visible to human reviewers (after they submit their own) and area chairs
- **No human reviewers or decisions are being replaced by AI**
- Authors may address AI-generated reviews in their rebuttal

## 10. Dual Submission Policy

AAAI's dual-submission policy is strict and actively enforced:

### At Submission Time
- AAAI-26 **will not consider** any paper that is:
  - Under review at another archival venue (journal or conference)
  - Already published or accepted at an archival venue
- **Workshops** and **preprint servers** (e.g., arXiv) are **acceptable** and do not count as archival venues
- Authors may retract a submission from a concurrent venue (e.g., NeurIPS-25) and submit to AAAI-26, provided the retraction occurs **before** the AAAI-26 submission
- Authors must **confirm** at submission time that the paper is not under review elsewhere

### After Submission
- Once submitted to AAAI-26, authors **may not submit the same paper** to another archival venue until they receive an accept/reject decision or withdraw from AAAI-26

### Enforcement
- If a concern is raised about overlapping submissions, **at least three people** will inspect both submissions
- If all agree there is "excessive technical overlap," the paper will be **summarily rejected** and organizers of the other venue will be notified
- Summary rejections are **final**

**Sources:**
- [AAAI-26 Submission Instructions](https://aaai.org/conference/aaai/aaai-26/submission-instructions/)

## 11. Preprint Policy

- **Preprints on arXiv (and similar servers) are allowed.** They are explicitly listed as acceptable and not considered archival venues.
- However, posting a preprint may reveal author identity, which is in tension with double-blind review. AAAI does not prohibit this but authors should be aware.
- Reviewers are "excused for not knowing about all non-refereed work (e.g., appearing on arXiv)," but if prior arXiv work is "widely known in the field, its existence may be considered by reviewers in assessing a submission's novelty."
- Papers published less than **two months before August 1, 2025** are considered "contemporaneous" -- authors are not obliged to address them (but encouraged to do so in camera-ready).
- **After acceptance:** authors may modify the paper title until the preprint deadline (**November 13, 2025**) by emailing AAAI. After that date, changes require publisher request.

## 12. Supplementary Materials Rules

Supplementary material is **optional** and due by **August 4, 2025** (3 days after the paper deadline). Uploaded via OpenReview.

### Three Types Allowed

| Type | Format | Purpose |
|------|--------|---------|
| Technical Appendix | PDF | Proofs, experimental details, dataset descriptions, additional results |
| Multimedia Appendix | ZIP | Images, audio, video demonstrations |
| Code & Data Appendix | ZIP | Source code and data for reproducibility |

### Key Rules
- **Reviewers are NOT obligated** to review supplementary material
- The main submission **must be self-contained** -- essential elements should be in the main paper
- **No web pointers** in supplementary material (violates blind review + no-modification policy)
- **Cannot be updated** after the August 4 deadline under any circumstances
- All supplementary material must be **properly anonymized**
- If dataset exceeds OpenReview size limit, upload a representative subset
- Authors are encouraged to share supplementary material upon acceptance

### What Goes Where
- Critical evaluation content --> main paper (7 pages)
- Additional proofs, experiments, dataset details --> Technical Appendix
- Demos, visualizations --> Multimedia Appendix
- Code for reproducibility --> Code & Data Appendix

## 13. Author Response / Rebuttal Period

### Timeline
- **October 7--13, 2025** -- Author feedback window (only for papers that passed Phase 1)

### Format
- **2,500 characters maximum** -- a single rebuttal responding to ALL reviews
- Markdown formatting is allowed
- **No URLs permitted** -- responses containing URLs may be withheld
- No additional files, code, diagrams, or new results can be uploaded

### What to Address
- Authors can respond to **all reviews**, including the AI-generated review
- Focus on **substance** of reviews, particularly **factually incorrect claims**
- Avoid focusing on tone, presentation, or perceived fairness
- **Do not provide new experimental results**, even if reviewers request them
- Clarifying why certain experiments are not needed is acceptable

### What NOT to Do
- Do not provide new results
- Do not include URLs
- Do not upload additional files (including the reproducibility checklist if it was missing)
- Do not update supplementary materials
- Complaints about reviews should go in the rebuttal, **not** to the Ethics Committee

### Review Process Context
- Phase 1: 2 human reviews + 1 AI-generated supplementary review
- Phase 1 rejections notified September 15; reviews (including AI) visible immediately
- Phase 2: Additional human reviews for papers not rejected in Phase 1
- After rebuttal: SPC recommendation based on all reviews, response, and reviewer discussion
- SPC has access to AI-generated "Discussion Summary Assistance"
- **Scores are NOT averaged** -- recommendations made by senior members considering all information

## 17. Conflict of Interest Rules

### For Reviewers
Reviewers must:
- **Not review submissions** from their own institution or close collaborators
- **Promptly disclose** any conflicts, including:
  - Prior co-authorship
  - Close professional relationships
  - Personal relationships
- **Exclude themselves** from the review process when conflicts exist
- **Maintain confidentiality** of the review process
- **Not use information** from the review process for personal advantage

### For Authors
- Authors must use the same OpenReview account across all submissions and review assignments
- All authors of AAAI-26 submissions are **expected to join the reviewer pool** (light load) unless extenuating circumstances apply
- Failure to meet reviewer expectations may result in sanctions such as paper rejection
- Paper matching uses multiple factors beyond bidding (area of expertise, publication history, geographic diversity) to minimize conflicts

### COI Enforcement
- The paper matching process uses "state-of-the-art algorithms with robustness checks against bid manipulation"
- Unsanctioned mutual bidding has "minimal effect" on paper matching
- AAAI is actively investigating potential ethics issues and confirmed violations will lead to consequences

## 18. Figure/Table Formatting

### Figures
- Must be `.jpg`, `.png`, or `.pdf` (no `.gif`, `.ps`, `.eps`)
- **300 dpi minimum** resolution
- Must be **cropped outside of LaTeX** -- do not use `\includegraphics` with `clip` or `trim` options
- Place figures at **top of page** (preferred) to save space
- Figures may span both columns using `figure*` environment
- Must not invade margins
- Use `\usepackage{graphicx}` (included in required preamble)
- Number figures sequentially
- **Do not use `minipage`** to group figures
- For `pgfplots`: create figures first, export as PDF, then import

### Figure Captions
- Must appear **under** the illustration
- 10pt roman font
- Not smaller, bold, or italic (individual words may be italicized for differentiation)

### Tables
- 10pt roman type (may be reduced to 9pt if necessary)
- **Do not use `\resizebox`** on entire tables
- For narrow columns, use `\setlength{\tabcolsep}{1mm}` or adapt content
- Tables too wide for one column must span both columns (`table*`)
- If still too wide, split into separate tables

### Table Captions
- Must appear **under** (not above) the table
- 10pt roman font

### Color Usage
- **Color restricted to figures only** -- never in text
- Must be **CMYK colorspace** (not RGB)
- Must be **WCAG 2.0 compliant** (contrast ratio > 4.5:1)
- Paper must be decipherable without color (archival version is B&W)
- Grayscale is preferred; red converts to black, yellow may disappear

### Algorithms
- Should appear floated to top or bottom of page
- Caption appears in the header, left-justified, enclosed between horizontal lines
- Algorithm body terminated with a horizontal line
- Use `algorithm` and `algorithmic` packages (recommended, not required)

### Listings (Code)
- Use the `listings` package with `newfloat`
- Caption in header, enclosed between horizontal lines
- No background color
- Line numbers must appear within text column

## 19. Bibliography System (.bst File)

### Required Setup
- Use **`aaai2026.bst`** bibliography style (included in Author Kit)
- The style file `aaai2026.sty` automatically sets `\bibliographystyle{aaai2026}` -- do NOT include a separate `\bibliographystyle` command
- Use **`natbib`** package (required in preamble, no options)

### Citation Format
- Author-year style: `(Author Year)`
- Single author: `(Newell 1980)`
- Two authors: `(Feigenbaum and Engelmore 1988)`
- Three authors: `(Ford, Hayes, and Glymour 1992)`
- Four or more: `(Ford et al. 1997)`

### Available Citation Commands
| Command | Output |
|---------|--------|
| `\cite{key}` | `(Author Year)` -- full citation |
| `\shortcite{key}` | `(Year)` -- year only |
| `\citeauthor{key}` | `Author` -- name only, no parens |
| `\citeyear{key}` | `Year` -- year only, no parens |

All `natbib` citation commands are also available.

### Reference Size
- References may be the same size as surrounding text
- If the paper exceeds page limits, reference font may be reduced to **`\small` (9pt)** but **no smaller** than 9pt with 10pt line spacing

### Important Notes
- `aaai2026.sty` is **incompatible with hyperref and navigator** -- using either will garble references
- References must flow directly after text without page breaks
- The `.bib` file is the only file that may be separate from the single `.tex` source

## 20. Pre-Submission Checklist

Use this checklist before submitting to AAAI-26.

### Content
- [ ] Paper is within **7 pages** of technical content (excluding references and checklist)
- [ ] Abstract is complete and informative (no placeholders)
- [ ] Title uses **Title Case** capitalization ([check here](https://titlecaseconverter.com/) with "Chicago" style)
- [ ] All significant prior work is cited
- [ ] Reproducibility checklist is completed and appended after references
- [ ] Acknowledgements are **removed** from the review submission
- [ ] No new results or claims added after abstract deadline that change the paper's scope

### Anonymization
- [ ] Author names and affiliations removed ("Anonymous Submission")
- [ ] `\usepackage[submission]{aaai2026}` used in preamble
- [ ] Self-references anonymized where they could reveal identity
- [ ] PDF metadata cleaned (no author info in document properties)
- [ ] No identifying information in supplementary materials (no university logos, author images, identifiable voices)

### Formatting
- [ ] Compiles in **PDFLaTeX** without errors
- [ ] US Letter paper size (8.5" x 11"), NOT A4
- [ ] Two-column format using `aaai2026.sty`
- [ ] Times/Nimbus font for body text (not Computer Modern)
- [ ] **No Type 3 fonts** in document or figures
- [ ] All fonts embedded in PDF
- [ ] No overfull boxes (check `.log` file)
- [ ] No text/equations/figures intruding into margins
- [ ] No forbidden packages used (see Section 6)
- [ ] No forbidden commands used (see Section 6)
- [ ] No embedded links or bookmarks
- [ ] No page numbers, headers, or footers
- [ ] Figures are `.jpg`, `.png`, or `.pdf` only
- [ ] Figures cropped externally (no `clip`/`trim` in `\includegraphics`)
- [ ] Figure captions below figures, table captions below tables
- [ ] Color only in figures, CMYK, WCAG 2.0 compliant
- [ ] Source is a **single `.tex` file** (no `\input`)
- [ ] Bibliography uses `aaai2026.bst`
- [ ] `\pdfinfo{/TemplateVersion (2026.1)}` present

### Policies
- [ ] Paper is **not** under review at another archival venue
- [ ] Paper has **not** been published or accepted at another archival venue
- [ ] Author count is final (no changes allowed after August 1)
- [ ] Each author has **10 or fewer** total submissions across all technical tracks
- [ ] All authors registered with the **same OpenReview account** they use for reviews
- [ ] Dual submission confirmation checked at submission time
- [ ] If using LLM-generated text, it is only as part of experimental analysis

### Supplementary Materials (if applicable)
- [ ] Due by **August 4, 2025** (3 days after paper deadline)
- [ ] Properly anonymized
- [ ] No web pointers included
- [ ] Technical appendix as PDF
- [ ] Multimedia and code as ZIP
- [ ] Content verifiable and complete (cannot be updated later)

### Final Checks
- [ ] PDF opens correctly and is compatible with Acrobat 5+
- [ ] All pages render correctly
- [ ] Grant numbers and acknowledgements are correct (for camera-ready only)
- [ ] Abstract submitted by **July 25** with full title
- [ ] Full paper uploaded by **August 1**
