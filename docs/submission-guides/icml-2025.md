## 2. Page Limits

### Submission Phase

| Component | Limit |
|---|---|
| **Main body** | **8 pages maximum** |
| References | Unlimited (does not count toward page limit) |
| Impact statement | Unlimited (does not count toward page limit) |
| Acknowledgements | **Omit during review** (does not count toward page limit) |
| Appendices | Unlimited pages (included in same PDF) |

### Camera-Ready Phase

| Component | Limit |
|---|---|
| **Main body** | **9 pages maximum** (one extra page vs. submission) |
| References | Unlimited |
| Impact statement | Unlimited (mandatory for main track, optional for position papers) |
| Acknowledgements | Unlimited (now permitted) |
| Appendices | Unlimited pages (included in same PDF) |

**What counts toward the 8/9-page limit:** All text, figures, tables, equations, and algorithms in the main body of the paper.

**What does NOT count:** References, impact statement, acknowledgements, and appendices (everything after the main body).

**Important:** Reviewers are not required to look beyond the first 8 pages (i.e., appendices). The main body must be self-contained.

**Source:** [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers) | [ICML 2025 Author Instructions](https://icml.cc/Conferences/2025/AuthorInstructions)

## 3. File Requirements

### Format
- **PDF only.** No Word, no other formats. Submissions in any format other than PDF will not be accepted.
- **LaTeX only** for typesetting (the style file is LaTeX-only; Word users must convert to PDF).
- Paper must be submitted as a **single file** (main body + references + appendices combined).

### File Size Limits

| Phase | Maximum Size |
|---|---|
| Submission | **50 MB** |
| Camera-ready | **20 MB** |
| Publication agreement (camera-ready) | 10 MB |

### Paper Size
- **US Letter** (8.5 x 11 inches). Not A4.
- The style file enforces this (`\paperwidth=8.5in`, `\paperheight=11in`).

### Font Requirements
- **10-point Times** typeface throughout the main text.
- Use **Type-1 fonts** only. Avoid Type-3 fonts (though the 2025 format checker does not enforce this).
- Use `pdflatex` rather than `latex` + `dvips` to avoid Type-3 font issues.
- The `microtype` package is recommended for better typesetting.

### PDF Metadata
- Do not use macOS Preview or similar tools to process PDFs (may alter margins).

**Source:** [ICML 2025 Author Instructions](https://icml.cc/Conferences/2025/AuthorInstructions)

## 4. Desk-Rejection Triggers

The following will cause your paper to be **rejected without review**:

1. **Formatting violations** -- Non-conformance to the style file, altered margins/spacing, page limit exceeded, wrong paper size.
2. **Anonymization violations** -- Author-identifying information in the submission (names, acknowledgements, grant numbers, non-anonymous code links).
3. **Dual submission violation** -- Paper substantially similar to one already published or under review at another venue.
4. **Reciprocal reviewing failure** -- No author designated as a reviewer, or designated reviewer fails to serve adequately. (Both 2025 and 2026; new for 2025.)
5. **Placeholder abstracts** -- Submitting a placeholder abstract and then substantially rewriting it after the abstract deadline.
6. **Author list changes after abstract deadline** -- Adding or removing authors without program chair approval (reordering is permitted).
7. **Prompt injection attempts** -- Explicitly noted as grounds for automatic desk rejection (2026).
8. **False conflict-of-interest declarations** -- Manipulating COI to influence reviewer assignment.

**Source:** [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers) | [ICML 2026 Call for Papers](https://icml.cc/Conferences/2026/CallForPapers)

## 5. Anonymization Rules

ICML uses **double-blind review**. Both of the following apply:

### Authors Must:
- **Remove all author-identifying information** from the title page and body.
- **Refer to their own prior work in the third person.** For example, write "Smith et al. (2024) showed..." rather than "In our prior work, we showed..."
- **Omit acknowledgements, grant numbers, and links to public code repositories** in the submission.
- **Remove author names from supplementary code/data** (including LICENSE files that may contain names).
- **Anonymize supplementary manuscripts** -- if referencing unpublished co-authored work, submit anonymized copies via OpenReview.

### Authors May:
- **Post preprints on arXiv** (see Section 11).
- **Give talks** about the submitted work during the review period.
- Reference their own published work by title in the bibliography (but refer to it in third person in the text).

### Authors Must NOT:
- Advertise or identify the work as an ICML submission during the review period.
- Include non-anonymized repository links (use anonymous GitHub branches or upload as supplementary material).
- Reference non-anonymized versions of the work in the submission itself.

**Consequence of violation:** Papers revealing author identity (explicitly or implicitly) may be rejected without review.

**Source:** [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers) | [ICML 2025 Author Instructions](https://icml.cc/Conferences/2025/AuthorInstructions)

## 6. Style File & Formatting

### Style Package

Download the official LaTeX style files:

- **ICML 2025:** [icml2025.zip](https://media.icml.cc/Conferences/ICML2025/Styles/icml2025.zip)
- **ICML 2026:** Available at the ICML 2026 Styles page on [icml.cc](https://icml.cc/Conferences/2026/AuthorInstructions)

The zip contains:

| File | Purpose |
|---|---|
| `icml2025.sty` / `icml2026.sty` | Main style file (do not modify) |
| `icml2025.bst` / `icml2026.bst` | BibTeX style file |
| `example_paper.tex` | Example/template paper |
| `example_paper.bib` | Example bibliography |
| `fancyhdr.sty` | Running headers |
| `algorithm.sty` | Algorithm environment |
| `algorithmic.sty` | Pseudocode formatting |

### Usage

```latex
% For submission (blind review):
\usepackage{icml2025}

% For camera-ready (accepted):
\usepackage[accepted]{icml2025}

% If hyperref causes build issues:
\usepackage[nohyperref]{icml2025}
```

### Page Layout (enforced by style file)

| Property | Value |
|---|---|
| Paper size | US Letter (8.5 x 11 in) |
| Columns | Two-column |
| Text width | 6.75 inches |
| Text height | 9.0 inches |
| Column separation | 0.25 inches |
| Left margin | 0.75 inches |
| Top margin | 1.0 inch |
| Body font | 10pt Times |
| Line spacing | 11pt vertical spacing |

**Do not** include packages that alter layout (e.g., `geometry`, `savetrees`, `fullpage`). The style file detects and flags margin tampering at compile time.

### Section Heading Formatting

| Level | Size | Style | Spacing Before/After |
|---|---|---|---|
| Section | 11pt | Bold, content words capitalized | 0.25in / 0.15in |
| Subsection | 10pt | Bold, content words capitalized | 0.20in / 0.13in |
| Subsubsection | 10pt | Small caps, content words capitalized | 0.18in / 0.10in |

Use no more than three levels of headings.

### Title Formatting
- 14pt bold, centered between two 1pt horizontal rules.
- Capitalize content words; do **not** use ALL CAPS.
- Example: "Deep Learning for Artificial Intelligence" (correct)
- Not: "DEEP LEARNING FOR ARTIFICIAL INTELLIGENCE" (incorrect)
- If the title is too long for the running header, use `\icmltitlerunning{...}` before `\begin{document}`.

### Abstract
- Single paragraph, 4-6 sentences.
- Left column, 0.4 inches below author block.
- 10pt type, 11pt spacing, indented 0.25 inches on both sides.

### Recommended Packages
```latex
\usepackage{microtype}       % Better typesetting
\usepackage{graphicx}        % Figures
\usepackage{subfigure}       % Sub-figures
\usepackage{booktabs}        % Professional tables
\usepackage{hyperref}        % Clickable links
\usepackage{amsmath}         % Math
\usepackage{amssymb}         % Math symbols
\usepackage{mathtools}       % Extended math
\usepackage{amsthm}          % Theorems
\usepackage[capitalize,noabbrev]{cleveref}  % Cross-references
```

### Format Checker (Camera-Ready)
Run the [ICML format checker](https://papercheck.icml.cc/papercheck.html) on your camera-ready PDF. It validates:
- Compiled with latest ICML style file (unmodified)
- Correct title on first page
- Non-blank running title on subsequent pages
- Main body does not exceed 9 pages (excluding acknowledgements, impact statement, references)
- Title case formatting (not ALL CAPS)
- Abstract is single paragraph

The checker returns a **5-letter submission code** required for the camera-ready submission form.

**Source:** [ICML 2025 Author Instructions](https://icml.cc/Conferences/2025/AuthorInstructions) | [ICML style file](https://media.icml.cc/Conferences/ICML2025/Styles/icml2025.zip)

## 7. Contribution Types / Tracks

### Main Research Track
Original, rigorous research of significant interest to the machine learning community. Topics span:
- General machine learning
- Deep learning
- Evaluation and benchmarks
- Theory
- Systems and optimization
- Probabilistic methods
- Reinforcement learning
- Trustworthy ML
- Application-driven ML (healthcare, physical sciences, biosciences, sustainability)

**Application-driven papers** receive modified review consideration: novel method combinations for real-world tasks, non-standard datasets with proper documentation, and novel framing/evaluation are valued.

### Position Paper Track
Meta-level arguments about what *should* be done in machine learning (policy, practice, deployment, ethics).

| Aspect | Main Track | Position Track |
|---|---|---|
| Submission site | Main OpenReview | Separate OpenReview |
| Page limit | 8 pages | 8 pages |
| Camera-ready | 9 pages | 9 pages |
| Impact statement | **Required** | **Optional** |
| Lay summary (2026) | **Required** | **Not required** |
| Reciprocal reviewing | **Required** | **Not required** |
| Title prefix | None | Must start with "Position:" |
| Abstract | Standard | Must identify as position paper |
| Required sections | Standard | "Alternative Views" section (in main body) |
| Recommended sections | N/A | "Call to Action" section (2026) |

Position paper review criteria: position clarity, evidence quality (1-4), significance (1-4), discussion potential (1-4), argument clarity (1-4), related work coverage (1-4), and reviewer confidence (1-5).

**Source:** [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers) | [ICML 2025 Call for Position Papers](https://icml.cc/Conferences/2025/CallForPositionPapers)

## 8. Paper Checklist

ICML does not have a formal NeurIPS-style paper checklist that must be filled out and included in the submission PDF. However, the following are effectively required items that reviewers and area chairs evaluate:

### Claims and Evidence
- All claims must be clearly stated and supported by reproducible experiments and/or sound theoretical analysis.
- Methods and evaluation criteria must fit the problem.
- Theoretical proofs must be verifiable.
- Experimental designs must be sound and reproducible.

### Prior Work
- All relevant prior work must be acknowledged and differentiated from.
- Replace arXiv citations with peer-reviewed versions where available.
- Self-citations must use third-person language.

### Impact Statement
- **Required** (main track). Must appear before References in an unnumbered section.
- A minimal acceptable statement: "This paper presents work whose goal is to advance the field of Machine Learning. There are many potential societal consequences of our work, none which we feel must be specifically highlighted here."

### Lay Summary (2026 onward)
- Required for accepted papers at the camera-ready stage.
- Plain-language summary for public accessibility.

### Reproducibility
- Authors are encouraged to submit code.
- Reproducibility is considered during review.

### Accessibility
- Color-blind and visually impaired accessible figures.
- Inclusive, respectful language.
- Current bibliography with updated author names.

**Source:** [ICML 2025 Author Instructions](https://icml.cc/Conferences/2025/AuthorInstructions) | [ICML 2025 Reviewer Instructions](https://icml.cc/Conferences/2025/ReviewerInstructions)

## 9. LLM / Agent Usage Policy

### For Authors
- Authors **may** use LLMs and generative AI tools to assist with writing and research.
- Authors **must take full responsibility** for all content in the paper, including AI-generated content.
- AI-generated content is subject to the same standards as human-authored content regarding plagiarism and misconduct.
- Authors are **encouraged to explain** notable AI tool usage in their methodology section.
- **LLMs are not eligible for authorship.** Do not list an LLM as a co-author.

### For Reviewers
- Reviewers are **strictly prohibited** from using generative AI to write reviews.
- Reviewers must not input submission content into AI tools (submission content is privileged information).

**Source:** [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers) | [ICML 2025 Peer Review FAQ](https://icml.cc/Conferences/2025/PeerReviewFAQ)

## 10. Dual Submission Policy

### Prohibited
- Submitting a paper that is **identical or substantially similar** to one that has been previously published, accepted for publication, or is under review at another conference or journal.
- Submitting to multiple conferences simultaneously with overlapping author lists (concurrent ICML submissions with overlapping authors are treated as prior work, potentially justifying rejection on incremental grounds).

### Permitted
- Workshop presentations (without formal proceedings) do not constitute prior publication.
- Preprints on arXiv or similar servers are permitted (see Section 11).

### Key Nuance
"Any concurrent ICML submissions with an overlapping set of authors will also be treated as prior work." This means if you submit two related papers to the same ICML, each will be evaluated for whether it is incremental relative to the other.

**Source:** [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers)

## 11. Preprint Policy

- Authors **are allowed** to post versions of their work on preprint servers such as arXiv.
- Authors **are allowed** to give talks on submitted work during the review period.
- Authors **must not** advertise the work as being under ICML submission during the review period.
- The submission itself **must not** reference non-anonymized preprint versions.

This is a permissive policy -- arXiv posting is fully compatible with ICML submission.

**Source:** [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers)

## 12. Supplementary Materials

### Two Types Permitted

1. **Supplementary manuscripts** -- For anonymous references cited in the paper (e.g., your own concurrent unpublished work). Must be anonymized.
2. **Code and data** -- Zip files or PDFs. Anonymous GitHub branches are acceptable if the branch remains unmodified after the submission deadline.

### Rules
- All supplementary materials must be **anonymized** (remove author names, LICENSE files with names, etc.).
- Supplementary materials are **not published or archived** by the conference.
- There is **no separate deadline** -- supplementary materials are submitted alongside the paper.
- Traditional appendices belong in the **main PDF**, not as separate supplementary files. Reviewers will not look for appendices as a separate file.
- Reviewers are **not required** to examine supplementary materials.
- **Camera-ready:** There is no option to provide supplementary material with the final published version (2026). Code links should be included in the paper text instead.

### Important
The main paper (first 8 pages) must be self-contained. Do not rely on supplementary materials to convey essential content -- reviewers may not look at them.

**Source:** [ICML 2025 Author Instructions](https://icml.cc/Conferences/2025/AuthorInstructions) | [ICML 2026 Author Instructions](https://icml.cc/Conferences/2026/AuthorInstructions)

## 13. Author Response / Rebuttal

### Timeline
- **ICML 2025:** Author response period ends March 31, 2025 AoE.
- **ICML 2026:** Author response period ends March 30, 2026 AoE. AC-Reviewer discussion follows March 31 -- April 12.

### Rules
- Responses address factual errors and reviewer questions.
- **No revised papers** may be uploaded during rebuttal.
- All responses must maintain **anonymity** -- no non-anonymized URLs, no personal website links, no shortened URLs.
- Links are restricted to figures, tables, or code only.
- Reviewers cannot see responses until after the response period closes (2025).
- "Keep in mind that there is no need to respond to every minor question."
- Organize responses by reviewer ID.
- For accepted papers, author responses are published alongside reviews.

### Review Visibility
- **Accepted papers:** Reviews, meta-reviews, author responses, and reviewer-author discussions are made publicly available (2026: both original submission and camera-ready versions are published).
- **Rejected papers:** Reviews remain private by default. Authors may opt in to public review publication.

**Source:** [ICML 2025 Author Instructions](https://icml.cc/Conferences/2025/AuthorInstructions) | [ICML 2025 Peer Review FAQ](https://icml.cc/Conferences/2025/PeerReviewFAQ)

## 14. Camera-Ready Requirements

### Mandatory Steps
1. **Register** for the conference with the "Conference" option checked (Virtual Pass alone is not sufficient -- 2025).
2. One author must **sign the ICML Publishing Release form**.
3. The presenting author must **sign the Recording Release form**.
4. The corresponding author must **sign the PMLR agreement** and upload it as a PDF.
5. **Include the mandatory impact statement** (main track) before References.
6. **Complete the OpenReview camera-ready form.**
7. Enter a **lay summary** in the OpenReview form (required from 2025 onward).

### Technical Requirements
- Switch to `\usepackage[accepted]{icml2025}` in your LaTeX preamble.
- Main body: **9 pages maximum** (one extra page over submission).
- File size: **20 MB maximum**.
- Paper size: **US Letter** (not A4).
- Use **vector graphics** (EPS/PDF) for plots; bitmap (PNG) only for raster visualizations.
- Run the [ICML format checker](https://papercheck.icml.cc/papercheck.html) and obtain the 5-letter submission code.
- Title: capitalize content words, do not use ALL CAPS.
- Abstract: single paragraph, 4-6 sentences.
- Author names must match exactly between TeX file and OpenReview form.
- Use `\printAffiliationsAndNotice{\icmlEqualContribution}` or `\printAffiliationsAndNotice{}` as appropriate.
- Include code URL in the paper and optionally in the OpenReview form.

### Post-Conference Revision (2026)
After the conference, authors may (optionally) submit a revised camera-ready version incorporating feedback received during the event. Essential content must remain unchanged from the reviewed version.

### No Author Additions
Author list cannot be changed after the abstract deadline. Reordering is permitted.

**Source:** [ICML 2025 Author Instructions](https://icml.cc/Conferences/2025/AuthorInstructions)

## 15. Presentation Requirements

### ICML 2025
- **At least one author** must attend the conference **in person** to present the paper, barring exceptional circumstances.
- Accepted papers are presented as **posters** (poster session selection deadline: June 27, 2025).
- A **video upload** is due by June 15, 2025.

### ICML 2026 (Changed)
- In-person attendance is **no longer required**. Authors may choose to:
  - Present in person, or
  - Have papers included in proceedings only (requires virtual registration for at least one author).
- All accepted papers receive **equivalent treatment and award eligibility** regardless of presentation choice.

**Source:** [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers) | [ICML 2026 Call for Papers](https://icml.cc/Conferences/2026/CallForPapers)

## 16. Ethics Review Process

### Flagging
Reviewers and Area Chairs may flag submissions for ethics review during the review process. They select specific expertise areas relevant to the concern.

### Ethics Committee Role
- The ethics committee provides **comments and recommendations** but **cannot directly reject** a paper.
- **Program chairs** may reject papers on ethical grounds regardless of scientific quality.

### Categories Evaluated

| Category | Focus |
|---|---|
| Discrimination / Bias / Fairness | Applications where bias is relevant (e.g., hiring algorithms) |
| Inappropriate Applications & Human Rights | Work with direct human rights implications (e.g., weapons) |
| Responsible Research Practice | IRB approval or exemption for human subjects research |
| Privacy & Security | Ensuring individuals are not identifiable in research outputs |
| Legal Compliance | EU AI Act compliance, copyrighted training data acknowledgement |
| Research Integrity | Plagiarism, collusion (escalated directly to Program Chairs) |

### EU AI Act Compliance
Authors must ensure their work does not involve prohibited AI practices including: subliminal manipulation, exploiting vulnerabilities, unfair individual classification, criminal risk prediction, untargeted facial recognition scraping, workplace emotion inference, biometric categorization, and real-time law enforcement biometric collection in public spaces.

### Financial Conflicts
Authors must disclose financial conflicts of interest to Program Chairs.

### Sanctions
Violations may result in:
- Rejection of the paper
- Naming in an ethics violator list shared with other organizations (NeurIPS, ICLR, AAAI, etc.)
- Investigation by the Oversight Committee

**Source:** [ICML 2025 Publication Ethics](https://icml.cc/Conferences/2025/PublicationEthics) | [ICML 2025 Call for Papers](https://icml.cc/Conferences/2025/CallForPapers)

## 17. Conflict of Interest Rules

### Types of Conflicts

**Domain Conflicts** (institutional):
- Current and past institutional affiliations from the **past three years**.
- Entered in the Education & Career History section of your OpenReview profile.
- Use domain conflicts judiciously -- only for genuine conflicts with an institution.

**Personal Conflicts:**
1. **Family or close personal relationship** -- Direct personal connections that could bias review.
2. **PhD advisor/advisee relationship** -- Hierarchical academic mentorship (lifetime conflict).
3. **Current, frequent, or recent collaboration** -- "Recent" means within the **past three years**. Co-authors already in your OpenReview publication history do not require separate entries.

### Special Restrictions
- Authors may restrict visibility of sensitive conflicts to Program Chairs only if they would "significantly compromise the fairness of the review process."
- **False declarations** to manipulate reviewer assignment may result in **rejection of all submitted papers without review**.

### Profile Requirements
All authors must have OpenReview accounts with up-to-date profiles (Relations, History, Emails sections). Institutional email accounts activate immediately; non-institutional emails may take up to 2 weeks.

**Source:** [ICML 2025 COI Definitions](https://icml.cc/Conferences/2025/ConflictOfInterestDefinitions) | [ICML 2026 COI Definitions](https://icml.cc/Conferences/2026/ConflictOfInterestDefinitions)

## 18. Figure / Table Formatting

### Figures
- **Captions below** the figure. Do not include titles inside the graphic file itself.
- Center all artwork. Lines must be **at least 0.5pt thick** for reproduction.
- Text should not appear on a gray background.
- Label all distinct components. Graphs need axis labels and legends.
- Number sequentially. Caption in **9pt** type, centered (flush left if multi-line).
- **At least 0.1 inches** of space before and after the caption.
- Float to top or bottom of column. Wide figures use `figure*` environment.
- Two-column figures must be placed at top or bottom of page.

### Graphics Format
- **Vector formats** (EPS, PDF) for plots and line art.
- **Lossless bitmap** (PNG) for raster graphics with sharp lines.
- **JPEG** for photo-like images.
- Keep file sizes reasonable.

### Tables
- **Captions above** the table (opposite of figures).
- Caption in **9pt** type, centered (flush left if multi-line).
- **At least 0.1 inches** of space before and after the caption.
- Use the `booktabs` package for professional-looking tables (`\toprule`, `\midrule`, `\bottomrule`).
- Float to top or bottom of column. Wide tables use `table*` environment.

### Accessibility
- Use colorblind-friendly palettes.
- Ensure figures are legible for people with visual impairments.
- Do not rely solely on color to convey information.

### Example Table Structure
```latex
\begin{table}[t]
\caption{Your caption here.}
\label{tab:example}
\vskip 0.15in
\begin{center}
\begin{small}
\begin{sc}
\begin{tabular}{lcccr}
\toprule
Column 1 & Column 2 & Column 3 \\
\midrule
Row 1 & value & value \\
Row 2 & value & value \\
\bottomrule
\end{tabular}
\end{sc}
\end{small}
\end{center}
\vskip -0.1in
\end{table}
```

**Source:** [ICML 2025 example_paper.tex](https://media.icml.cc/Conferences/ICML2025/Styles/icml2025.zip)

## 19. Bibliography System

### Required Setup
```latex
% In preamble (loaded automatically by icml2025.sty):
% \RequirePackage{natbib}   -- already loaded

% At end of document:
\bibliography{your_refs}
\bibliographystyle{icml2025}
```

- **natbib** is the required bibliography package (loaded by the style file).
- **icml2025.bst** / **icml2026.bst** is the required BibTeX style file.
- Do **not** use biblatex. The style file loads natbib automatically.

### Citation Commands

| Command | Output | Use Case |
|---|---|---|
| `\cite{key}` | (Author et al., 2024) | Parenthetical citation (remapped to `\citep`) |
| `\citep{key}` | (Author et al., 2024) | Parenthetical citation |
| `\citet{key}` | Author et al. (2024) | Textual citation |
| `\yrcite{key}` | (2024) | Year-only parenthetical (when author name is in text) |
| `\cite{k1,k2,k3}` | (A, 2020; B, 2021; C, 2022) | Multiple citations, chronological order |

### Bibliography Best Practices
- **Replace arXiv citations** with peer-reviewed versions where available.
- **Protect capital letters** in BibTeX titles: use `{B}ayesian`, `{L}ipschitz`, `{GPU}`.
- Citation font size must match main body text.
- Ensure up-to-date venue names and author attributions (use current author names).
- References should be **complete** (include page numbers, volume, DOI where possible).
- **Alphabetize** by first author surname; single-author entries precede multi-author.
- **APA reference format** regardless of formatter.

## Appendix: Review Criteria & Rating Scale

For reference, reviewers evaluate papers on the following scale:

| Score | Meaning |
|---|---|
| 5 | Strong accept |
| 4 | Accept |
| 3 | Weak accept (leaning toward acceptance) |
| 2 | Weak reject (leaning toward rejection) |
| 1 | Reject |

### Main Track Evaluation Dimensions
- **Summary:** Concise overview of contributions
- **Claims and Evidence:** Are claims supported? Methods appropriate?
- **Prior Work:** Connection to existing literature, missing citations
- **Originality, Significance, Clarity:** Core quality dimensions
- **Questions for Authors:** Issues that could change the evaluation
- **Ethical Issues:** Flagging for ethics review if needed

### Reciprocal Reviewing (2025+)

| Condition | Requirement |
|---|---|
| Every submission | At least one author designated as reciprocal reviewer |
| Author with 4+ submissions | Must serve as a reviewer |
| Reciprocal reviewer cap (2026) | Each author can be designated for at most 2 submissions |
| Exemptions | Authors serving as SAC, AC, or other organizing roles |

Reviewer qualifications: research experience equivalent to a second-year graduate student; primary author on at least two peer-reviewed papers from related venues (ICML, NeurIPS, ICLR, CVPR, ACL, JMLR, etc.).
