# COLM 2025 Submission Guide

Complete reference for submitting to COLM 2025 (Conference on Language Modeling). Sourced from the official [Call for Papers](https://colmweb.org/2025/cfp.html), [Author Guide](https://colmweb.org/2025/AuthorGuide.html), [Review Guidelines](https://colmweb.org/2025/ReviewGuide.html), [Code of Ethics](https://colmweb.org/2025/CoE.html), [Code of Conduct](https://colmweb.org/2025/CoC.html), [Conflict of Interest Policy](https://colmweb.org/2025/coi-policy.html), and the [LaTeX template repository](https://github.com/COLM-org/Template).

COLM is a newer venue (inaugurated 2024) focused exclusively on language modeling research. Single-track conference format.

Last updated: 2026-03-21.

## 2. Page Limits

| Content | Limit | Counts toward 9 pages? |
|---------|-------|------------------------|
| Main text (including figures, tables) | **9 pages** (strict maximum) | Yes |
| References / bibliography | Unlimited | No |
| Appendices | Unlimited | No |
| Ethics statement (optional) | Up to 1 page | No |
| Reproducibility statement (optional) | Up to 1 page | No |

Reviewers are **not required** to read appendices. Any content critical to evaluation should be in the main 9 pages.

## 3. File Requirements

- **Format:** Single PDF, generated via `pdflatex` (or `dvips`/`ps2pdf` with `-t letter` flag)
- **Paper size:** US Letter (8.5 x 11 inches) -- not A4
- **LaTeX template:** Must use the official COLM style files from [github.com/COLM-org/Template](https://github.com/COLM-org/Template)
- **Key template files:**
  - `colm2025_conference.sty` -- main style file
  - `colm2025_conference.bst` -- bibliography style
  - `colm2025_conference.tex` -- template/sample document
  - `math_commands.tex` -- mathematical command definitions
  - `fancyhdr.sty` and `natbib.sty` -- bundled dependency packages
- **Supplementary materials:** Upload as zipped files (max 50 MB) or via anonymous URLs; must be self-contained and anonymized
- **Figures:** Vector formats (PDF/SVG) preferred; minimum font size "small"; must be readable in black and white
- **No Microsoft Word submissions** -- LaTeX only

### Style file options

```latex
\usepackage{colm2025_conference}             % submission (default, includes line numbers)
\usepackage[final]{colm2025_conference}      % camera-ready (accepted papers only)
\usepackage[preprint]{colm2025_conference}   % preprint version
```

Source: [Template repository](https://github.com/COLM-org/Template) (commit `63933e03`, December 2024)

## 4. Desk-Rejection Triggers

COLM does not publish an explicit desk-rejection checklist, but the following will result in rejection without review based on stated policies:

1. **Exceeding the 9-page main text limit** -- described as a "strict upper limit"
2. **Non-anonymous submissions** -- "Non-anonymous submissions will be rejected without review"
3. **Modifying the style file** -- "Do not change any aspects of the formatting parameters in the style files" (explicitly stated as grounds for rejection)
4. **Missing or incomplete OpenReview profiles** -- all authors must have updated profiles by the abstract deadline
5. **Dual submission to another archival venue** while under COLM review
6. **False conflict-of-interest declarations** -- "may result in rejection without review"
7. **Placeholder or duplicate abstracts** -- will be removed from the system

Sources: [CFP](https://colmweb.org/2025/cfp.html), [Author Guide](https://colmweb.org/2025/AuthorGuide.html), [COI Policy](https://colmweb.org/2025/coi-policy.html), [Template](https://github.com/COLM-org/Template)

## 5. Anonymization Rules (Double-Blind)

COLM uses **double-blind review**. During review, submissions are visible only to the assigned program committee -- not to the general public.

**Required:**
- Remove all author names and affiliations (handled automatically when the `\colmfinalcopy` macro remains commented out)
- Remove acknowledgments sections
- Remove any links (e.g., GitHub URLs) that would reveal author identity
- Anonymize all supplementary code and data
- Verify that author names do not appear anywhere in the submitted PDF

**Self-citation guidance:** The template and policies do not provide explicit self-citation wording (unlike NeurIPS's "third person" rule), but the double-blind requirement implies authors should avoid self-identifying language.

**Post-decision visibility:**
- Accepted papers: reviews and papers made public on OpenReview
- Rejected/withdrawn papers: remain private and unpublished

Source: [CFP](https://colmweb.org/2025/cfp.html)

## 6. Style File and Formatting Specifications

### Page layout
| Parameter | Value |
|-----------|-------|
| Text width | 5.5 inches |
| Text height | 9.0 inches |
| Left margin | 1.5 inches |
| Top margin | 1 inch from page top |
| Paper size | US Letter (8.5 x 11 in) |

### Typography
| Element | Specification |
|---------|---------------|
| Body font | Palatino (mandatory), 10pt on 11pt leading |
| Monospace font | Inconsolata |
| Title | 17pt, left-aligned |
| Section headings | 12pt bold, lowercase (except first word/proper nouns) |
| Subsection headings | 10pt bold |
| Subsubsection headings | 10pt bold italic |
| Abstract | 10pt, indented 0.5 in on both sides, one paragraph |
| Paragraphs | 1/2 line space between, no indentation |

### Citations
- Use `natbib` package with author-year format
- `\citet{}` for in-text narrative citations (unparenthesized)
- `\citep{}` for parenthetical citations
- References listed alphabetically by first author

### Headers
- Submission mode: "Under review as a conference paper at COLM 2025"
- Preprint mode: "Preprint. Under review."
- Final mode: "Published as a conference paper at COLM 2025"

### Restrictions
- Do **not** modify formatting parameters in the style file (width, length, font sizes)
- Do **not** use the `geometry` package unless with the `pass` flag
- Do **not** override defined styles with custom headers
- Do **not** use hand-drawn figures
- All pages must be numbered
- Line numbers are included automatically in submission mode (do not cite them in the paper text)

Source: [Template .tex and .sty files](https://github.com/COLM-org/Template)

## 7. Tracks / Paper Types

COLM is a **single-track** conference. There is no multi-track system or separate paper type selection (unlike NeurIPS).

### Topic areas (18 categories)

1. Alignment (fine-tuning, instruction-tuning, RLHF, prompt tuning)
2. Data (pre-training data, alignment data, synthetic data analysis/curation)
3. Evaluation (benchmarks, metrics, oversight, protocols)
4. Societal implications (bias, equity, misuse, climate impact)
5. Safety (security, privacy, misinformation, adversarial robustness)
6. Science of LMs (scaling laws, emergent capabilities, interpretability)
7. Compute-efficient methods (distillation, compression, quantization)
8. Engineering (distributed training/inference, optimization)
9. Learning algorithms (learning, unlearning, meta-learning, continual learning)
10. Inference algorithms (decoding, reasoning, search, planning)
11. Cognitive and philosophical perspectives (linguistics, neuroscience, philosophy, law)
12. Multilingual and low-resource LMs (vernacular languages, multiculturalism)
13. Knowledge and grounding (factuality, retrieval-augmented generation, world models)
14. Multimodal LMs (audio, vision, other modalities)
15. Embodied LMs (robotics, action, perception)
16. Interactive systems (conversation, multi-agent learning)
17. Tools and code integration (software engineering, APIs)
18. Domain-specific applications (visual, code, math, chemistry, medicine, education)

**Presentation format:** Mix of oral presentations and posters.

Source: [CFP](https://colmweb.org/2025/cfp.html)

## 8. Checklist Requirements

COLM 2025 does **not** mandate a formal paper checklist (unlike NeurIPS, which requires a 16-question checklist). However, the following are effectively required or strongly encouraged:

- **Ethics statement** (optional, up to 1 page, does not count toward limit) -- encouraged for papers with ethical considerations
- **Reproducibility statement** (optional, up to 1 page, does not count toward limit) -- authors should include "hyperparameters used, clear explanations of any assumptions and a complete proof of the claims, a complete description of data processing steps"
- **Code of Ethics acknowledgment** -- all authors must "explicitly acknowledge" the Code of Ethics during the OpenReview submission process

Source: [Author Guide](https://colmweb.org/2025/AuthorGuide.html)

## 9. LLM Usage Policy

COLM follows **arXiv's guidelines** on generative AI tools, with additional COLM-specific requirements.

### arXiv's baseline policy (adopted by COLM)
- Authors must **report any significant use** of generative AI language tools
- **LLMs cannot be listed as authors** -- only humans who take full responsibility may be authors
- Authors remain **fully responsible** for all content, including any errors or misleading information generated by AI tools

### COLM-specific disclosure requirements (adapted from ICLR)

**Must disclose:**
- Using LLMs to originate research ideas
- Using LLMs to write original content
- Using LLMs to generate data or plots
- Using LLMs to conduct evaluation

**Does not require disclosure:**
- Grammar-checking and typo-fixing tools
- AI coding assistants (e.g., GitHub Copilot)

### Prohibitions
- Must not "deliberately make false or misleading claims, fabricate or falsify data, or misrepresent results"

Sources: [CFP](https://colmweb.org/cfp.html), [arXiv AI Policy](https://blog.arxiv.org/2023/01/31/arxiv-announces-new-policy-on-chatgpt-and-similar-tools/)

## 10. Dual Submission Policy

"Full papers submitted to COLM (i.e., those in the review process after the abstract deadline has passed) cannot be under review for another conference simultaneously."

**Key rules:**
- Authors must withdraw papers from other venues **before** the COLM full paper submission deadline, or withdraw from COLM before submitting elsewhere
- The policy applies throughout the entire review period (March 28 -- July 7, 2025)
- **Non-archival workshop submissions** do not trigger this restriction
- Papers can be revised multiple times up to the paper submission deadline
- "Changes to the paper will not be allowed while the paper is being reviewed"

Source: [CFP](https://colmweb.org/2025/cfp.html)

## 11. Preprint Policy

COLM does not publish an explicit, detailed preprint policy document separate from the CFP. Based on available guidance:

- The LaTeX template includes a `[preprint]` option that sets the header to "Preprint. Under review."
- There is no stated prohibition on posting preprints to arXiv or other repositories
- The double-blind review process means reviewers should not know author identities; however, COLM does not appear to have a formal "no searching for preprints" instruction for reviewers
- Use the `[preprint]` style option (not `[final]`) if posting a preprint

**Recommendation:** Post preprints with the `[preprint]` option. Avoid writing "Under review at COLM 2025" on the preprint. As with similar venues, avoid aggressive social media promotion that could compromise double-blind review.

Source: [Template .sty file](https://github.com/COLM-org/Template)

## 12. Supplementary Materials

- **Upload:** As part of the OpenReview submission -- either as zipped files (max **50 MB**) or via anonymous URLs
- **Content:** Source code, data, additional experiments, proofs
- **Anonymization:** All supplementary material must be self-contained and anonymized
- **Format:** Submit as a single file combining paper, appendices, and supplementary text, with supplementary material "clearly marked"
- **Reviewer obligation:** Reviewers are "encouraged but not required" to review supplementary materials
- **Appendices** (unlimited pages) do not count toward the 9-page limit but reviewers need not read them

Source: [Author Guide](https://colmweb.org/2025/AuthorGuide.html)

## 15. Review Process

### Platform
All submissions and reviews are handled through **OpenReview** at the COLM 2025 Conference group.

### Structure
- **Double-blind** review -- author identities hidden from reviewers and area chairs
- **Single-track** conference -- all papers compete in the same pool
- Reviews and author responses are initially non-public
- Submissions are visible only to assigned program committee members during review
- No public comments are solicited during the review period

### Review criteria (5 evaluation dimensions)
Reviewers assess papers holistically across these dimensions:

1. **Empiricism, Data, and Evaluation** -- natural data, strong experimental design, validated metrics
2. **Technological Impact** -- high-quality resources and artifacts enabling future research
3. **Ambition and Vision** -- forward-looking work that extends current boundaries
4. **Understanding Depth** -- principled modeling, careful analysis
5. **Clarity and Honesty** -- clear writing, balanced presentation, released materials

**Key principle:** "Weakness in some aspects cannot be an immediate reason to reject a paper." Reviewers are told to consider that "most researchers do not have access to large-scale compute."

COLM does not publish a numerical rating scale in its public reviewer guidelines, though OpenReview typically uses standard 1-10 scales.

### Reciprocal reviewing
COLM implements a two-part reciprocal reviewing system:
- **Per-submission:** Each submission must designate one author as a potential reviewer (or claim exemption)
- **Per-reviewer:** Authors with 4 or more submissions are automatically added to the reviewer pool, unless already on the program committee or COLM organization

### Post-decision
- **Accepted papers:** Papers and reviews are published on OpenReview
- **Rejected/withdrawn papers:** Remain private and unpublished

### Area chairs
- Papers assigned automatically via matching algorithm (no AC bidding)
- ACs manage reviewer quality, lead discussions, write meta-reviews, and recommend decisions
- AC guidelines adapted from NeurIPS 2024

Sources: [CFP](https://colmweb.org/2025/cfp.html), [Review Guidelines](https://colmweb.org/2025/ReviewGuide.html), [AC Guidelines](https://colmweb.org/2025/ac-guidelines.html)

## 16. Ethics Requirements

### Code of Ethics (8 principles)
All authors must read and acknowledge the [Code of Ethics](https://colmweb.org/2025/CoE.html) during submission. The 8 general ethical principles are:

1. Contribute to society and human well-being
2. Uphold high standards of scientific excellence
3. Avoid harm (including unintended consequences and environmental damage)
4. Be honest, trustworthy, and transparent
5. Be fair and avoid discrimination
6. Respect intellectual work (proper attribution, copyright)
7. Respect privacy (data handling, re-identification prevention)
8. Honor confidentiality

### Ethics review process
- Reviewers and area chairs may **flag** submissions for ethics committee evaluation
- Ethics reviewers provide comments (shared with primary reviewers, ACs, and authors)
- Ethics reviewers **cannot reject papers independently**
- **Program chairs may reject papers on ethical grounds** regardless of scientific quality
- Authors may respond to ethics concerns during the rebuttal period

### IRB / human subjects
- Researchers must "assess and report" the need for "ethical approvals from an appropriate ethical review board" when human subjects participate directly or as annotators

### Ethics statement
- Optional, up to 1 page, does not count toward page limit
- Encouraged for papers with ethical considerations

### Code of Conduct
- COLM adopts the [ICLR Code of Conduct](https://colmweb.org/2025/CoC.html)
- Covers all participants (attendees, organizers, reviewers, speakers, sponsors)
- Prohibits harassment, bullying, discrimination, retaliation
- Enforcement: warnings, expulsion, bans, institutional reporting

Sources: [Code of Ethics](https://colmweb.org/2025/CoE.html), [Code of Conduct](https://colmweb.org/2025/CoC.html), [CFP](https://colmweb.org/2025/cfp.html)

## 17. Pre-Submission Checklist

Before submitting to COLM 2025, verify:

### Formatting
- [ ] Main text is within the **9-page strict limit** (figures and tables count)
- [ ] Using the official, **unmodified** `colm2025_conference.sty`
- [ ] Paper compiled with `pdflatex` on **US Letter** paper size
- [ ] Font is **Palatino** (loaded automatically by the style file)
- [ ] Line numbers are present (automatic in submission mode)
- [ ] No `geometry` package (unless with `pass` flag)
- [ ] No custom headers overriding the style file
- [ ] All pages are numbered

### Anonymization
- [ ] No author names, affiliations, or acknowledgments visible
- [ ] `\colmfinalcopy` macro is **commented out**
- [ ] No GitHub links, personal websites, or other identifying URLs
- [ ] Supplementary code and data are anonymized
- [ ] No self-identifying language in the text

### Content
- [ ] Abstract is a single paragraph, indented, 10pt font
- [ ] References use `natbib` with `\citet{}` and `\citep{}` commands
- [ ] Figures at top or bottom of pages, vector format preferred
- [ ] All figures readable in black and white
- [ ] No `\todo{}` or draft macros remaining
- [ ] Ethics statement included if applicable (up to 1 page, does not count)
- [ ] Reproducibility statement included if applicable (up to 1 page, does not count)

### Submission system
- [ ] **All authors** have updated OpenReview profiles (by abstract deadline)
- [ ] OpenReview profiles use institutional email (non-institutional may take 2 weeks to activate)
- [ ] Abstract submitted by **March 22, 2025** -- genuine and informative, not a placeholder
- [ ] Author list is final and triple-verified (no changes possible after submission deadline)
- [ ] Code of Ethics acknowledged during submission
- [ ] Reciprocal reviewing requirement addressed (reviewer designated or exemption claimed)

### Policy compliance
- [ ] Paper is **not** simultaneously under review at another archival venue
- [ ] Non-archival workshop submissions are clearly separate
- [ ] LLM usage disclosed if used for research ideas, original content, data generation, or evaluation
- [ ] No fabricated data or misleading claims
- [ ] Conflict of interest disclosures are accurate and complete in OpenReview

### Supplementary materials (if applicable)
- [ ] Zipped file is under **50 MB**
- [ ] Materials are self-contained and anonymized
- [ ] Clearly marked as supplementary
