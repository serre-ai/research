## 1. How ARR Works

ACL uses a **two-stage review system** that is fundamentally different from NeurIPS/ICML:

### Stage 1: Centralized Rolling Review (ARR)

1. Authors submit to an **ARR cycle** (runs every ~10 weeks as of May 2025).
2. A Senior Area Chair (SAC) assigns an Area Chair (AC); the AC assigns at least **3 reviewers**.
3. Reviewers write reviews; authors write a response; reviewers discuss; the AC writes a **meta-review**.
4. Reviews and meta-review are released to authors at cycle end.

### Stage 2: Commitment to a Venue

1. After receiving reviews, authors **commit** the reviewed paper to a specific venue (e.g., ACL 2025) by that venue's commitment deadline.
2. The venue's program committee reads the ARR reviews + meta-review and makes **accept/reject decisions independently**.
3. Papers not accepted at ACL main conference are automatically considered for **Findings of ACL**.

### Key implications

- You can revise and resubmit to a later ARR cycle if reviews are poor.
- You can commit the same reviewed paper to different venues (but not simultaneously).
- Reviews travel with the paper -- venue PCs see all ARR reviews.

**Sources:** [ARR How It Works](https://aclrollingreview.org/reviewing), [ARR CFP](https://aclrollingreview.org/cfp)

## 3. Paper Types (Long vs Short)

### Long Papers (up to 8 pages)

Substantial, complete research contributions. Expected to include thorough evaluation, analysis, and discussion.

### Short Papers (up to 4 pages)

Focused contributions: a negative result, a small but complete experiment, an opinion piece, an interesting application nugget, or a small focused contribution.

### Special Paper Types (ACL 2025)

Both long and short formats accept:
- **Negative findings** -- results that didn't work, with analysis of why
- **Survey papers** -- comprehensive reviews of a subfield
- **Resource papers** -- new datasets, annotation schemes, tools
- **Position papers** -- argued opinions about the field's direction
- **Reproduction studies** -- successful or failed reproductions
- **Novel linguistic insights** using existing techniques

### Theme Track: ACL 2025

**"Generalization of NLP Models"** -- a special track with its own session and Thematic Paper Award covering:
- Cross-lingual, cross-domain, cross-task, compositional, structural, and robustness generalization
- Evaluation methods for generalization
- LLM limitations in generalization

### Findings of ACL

Papers not accepted to the main conference are automatically considered for **Findings of ACL** (an online companion publication with lower acceptance bar but still peer-reviewed). Authors can opt out.

**Source:** [ACL 2025 CFP](https://2025.aclweb.org/calls/main_conference_papers/)

## 4. Page Limits

| | Submission | Camera-Ready |
|---|---|---|
| **Long paper** | 8 pages content | 9 pages content |
| **Short paper** | 4 pages content | 5 pages content |

**What counts toward the page limit:**
- All body text, figures, tables in the main paper

**What does NOT count toward the page limit:**
- Limitations section (mandatory, unlimited)
- Ethical considerations section (optional, unlimited)
- References (unlimited)
- Acknowledgments (camera-ready only; omit during review)
- Appendices (unlimited, but must be in same PDF)

**Exceeding page limits is an automatic desk rejection.**

## 6. Paper Size & Layout Specifications

> **Critical ACL-specific requirement:** ACL uses **A4 paper**, not US Letter. This is different from some ML conferences.

### Paper Dimensions

| Property | Value |
|---|---|
| **Paper size** | A4 (21 cm x 29.7 cm / 8.27" x 11.7") |
| **All margins** | 2.5 cm (top, bottom, left, right) |
| **Columns** | Two-column layout |
| **Column width** | 7.7 cm |
| **Column gap** | 0.6 cm |
| **Column height** | 24.7 cm |

### PDF Verification

Check your PDF is A4 with `pdfinfo`:
```
Page size:      595.276 x 841.89 pts
```

### Font Sizes

| Element | Size | Style |
|---|---|---|
| Paper title | 15 pt | Bold |
| Author names | 12 pt | Bold |
| Author affiliation | 12 pt | Regular |
| Section headings | 12 pt | Bold |
| Subsection headings | 11 pt | Bold |
| Body text | 11 pt | Regular |
| "Abstract" label | 12 pt | Bold |
| Abstract text | 10 pt | Regular |
| Captions | 10 pt | Regular |
| Bibliography | 10 pt | Regular |
| Footnotes | 9 pt | Regular |

### Additional Layout Rules

- Abstract: max 200 words, indented 0.6 cm on each side
- New paragraphs: indent 0.4 cm (except first paragraph of a section)
- Single-spaced throughout
- Sections numbered with Arabic numerals (1, 1.1, 1.2)
- Body text starts 7.5 cm from top of page
- Hyperlinks: dark blue (#000099), not underlined or boxed
- Figures: prefer vector formats (PDF, EPS); avoid low-resolution rasters
- Grayscale readability strongly encouraged (don't rely on color alone)
- Non-English text: include English translation and Latin transliteration
- All fonts must be **embedded** in the PDF (verify with `pdffonts` -- all should show "yes" under "emb")

**Source:** [ACL Formatting Guidelines](https://acl-org.github.io/ACLPUB/formatting.html)

## 7. Bibliography & DOI Requirements

### Bibliography System

- Use **natbib** with the provided `acl_natbib.bst` style file
- Use `\citep{}` for parenthetical and `\citet{}` for textual citations

### Formatting Rules

- "References" heading: unnumbered section
- Alphabetical order by first author surname
- Use **full author names**, not initials
- Follow Computational Linguistics or APA style consistently

### DOI Requirements

- **Include DOIs whenever possible** -- this is a requirement, not optional
- Use ACL Anthology URLs as a secondary option when DOIs are unavailable
- When a preprint has been superseded by a refereed publication, cite the refereed version

### Citation Rules

- Cite all relevant refereed publications
- Papers appearing <3 months before submission deadline are "contemporaneous" and do not require detailed comparative analysis
- Authors are excused for missing unpublished/concurrent work

**Source:** [ACL Formatting Guidelines](https://acl-org.github.io/ACLPUB/formatting.html)

## 9. Anonymization Rules

ACL uses **double-blind (two-way anonymized) review**. Violations result in desk rejection.

### Required

- Remove all author names and affiliations
- Remove acknowledgments section (add only in camera-ready)
- Use third-person for self-citations: "Smith (2023) showed..." NOT "We previously showed (Smith, 2023)..."
- Anonymize all supplementary materials and external links
- Use anonymous hosting for code (e.g., [Anonymous GitHub](https://anonymous.4open.science/))

### Prohibited

- Author-identifying information anywhere in the PDF or supplementary materials
- Links to non-anonymous repositories (GitHub with your username, personal websites)
- Dropbox, Google Drive, or other file-tracking services (they can reveal identity)
- References to unavailable documents that would require reviewer contact with authors
- Including author names in PDF metadata

### No Anonymity Period for Preprints

As of February 2024, there is **no anonymity period** -- you may post or discuss non-anonymous preprints during review. However, see the preprint policy (Section 10) for incentive differences.

**Source:** [ARR CFP](https://aclrollingreview.org/cfp)

## 10. Preprint Policy

ACL's preprint policy is **more permissive** than NeurIPS/ICML:

- **No anonymity period.** You may post, update, or discuss non-anonymous preprints (e.g., on arXiv) at any time during review.
- However, ACL **incentivizes anonymous submissions**: special paper awards and priority in acceptance decisions for borderline papers are reserved for submissions that remain anonymous.

### Binding Non-Preprint Option

- Upon submission, authors select their preprint status.
- Choosing **"binding non-preprint"** prohibits posting the paper as a preprint until review completes.
- Violating a binding non-preprint commitment results in **desk rejection**.

**Source:** [ARR CFP](https://aclrollingreview.org/cfp)

## 11. Dual Submission Policy

### ARR + Other Venues

- Papers **cannot be under review elsewhere** during ARR review.
- This includes journals (TACL), conferences (IJCAI, NeurIPS, ICML, AAAI), and workshops with published proceedings.
- Violations result in desk rejection.

### ARR + ARR

- Cannot submit the same paper to two ARR cycles simultaneously.
- Cannot resubmit while a previous version is still under active review.

### ARR + Commitment

- Papers with an active ARR review cannot simultaneously be committed to a venue.
- Papers committed to a venue cannot be resubmitted to ARR between commitment and decision.

### Commitment + Commitment

- Depends on the dual submission policies of the respective venues. Typically prohibited.

### Key Difference from NeurIPS/ICML

NeurIPS and ICML have a single submission deadline and prohibit concurrent submissions. ACL's rolling system means you must track when your paper leaves "under review" status (after meta-review release) before submitting or committing elsewhere.

**Source:** [ARR CFP](https://aclrollingreview.org/cfp)

## 12. Mandatory Limitations Section

**This section is REQUIRED. Missing it triggers desk rejection.**

### Requirements

- Place after the conclusion, before references
- Title: "Limitations"
- Does **not** count toward page limits (unlimited length)
- Must discuss the limitations of the work only (not introduce new methods or analysis)
- No page break between conclusion and limitations

### What to Include

- Strong assumptions and robustness to violations
- Scope constraints (datasets, languages, domains tested)
- Number of runs or seeds tested
- Computational constraints
- When analyzing bias, state your definition explicitly and cite relevant work

### Reviewer Instructions

Reviewers are instructed **not to penalize honest disclosure of limitations**. Transparency is rewarded.

> **ACL-specific:** NeurIPS and ICML do not mandate a limitations section (though NeurIPS has a broader impact statement). ACL's requirement is strictly enforced with desk rejection.

**Source:** [ARR CFP](https://aclrollingreview.org/cfp), [Responsible NLP Checklist](https://aclrollingreview.org/responsibleNLPresearch)

## 13. Responsible NLP Research Checklist

**Mandatory.** Incorrect, incomplete, or misleading responses can result in desk rejection (enforced since December 2024).

The checklist is completed in the OpenReview submission form (not in the paper PDF). It covers:

### A. For Every Submission
- **A1: Limitations** -- describe assumptions, scope constraints, bias definitions
- **A2: Potential risks** -- malicious uses, environmental impact, fairness, privacy, dual-use

### B. Scientific Artifacts
- **B1: Citations** -- cite all artifact creators with version numbers and URLs
- **B2: Licenses** -- state licenses (e.g., CC-BY 4.0) for each asset
- **B3: Intended use** -- verify consistent use, anonymization, data statements
- **B4: PII & offensive content** -- explain detection and mitigation methods
- **B5: Documentation** -- data statements, model cards, datasheets
- **B6: Statistics** -- example counts, train/dev/test splits

### C. Computational Experiments
- **C1: Compute details** -- parameter counts, GPU hours, infrastructure
- **C2: Experimental setup** -- hyperparameter search, model selection, best values
- **C3: Descriptive statistics** -- error bars, confidence intervals, multiple seeds
- **C4: Packages** -- cite all libraries with versions (NLTK, SpaCy, etc.)

### D. Human Annotators / Research Participants
- **D1: Instructions** -- complete instructions, screenshots, risk warnings
- **D2: Recruitment & payment** -- platform, payment amounts, wage adequacy
- **D3: Consent** -- how data use was communicated
- **D4: Ethics board** -- IRB approval/exemption (omit institution name during review)
- **D5: Demographics** -- annotator demographics, protected categories

### E. AI Assistants
- **E1: Disclosure** -- disclose use of ChatGPT, Copilot, or similar tools

### How to Answer

- Answer "Yes" with section references pointing to the paper, or "No" with justification
- Avoid marking "Yes" for non-applicable questions
- Quality of justification matters

**Source:** [Responsible NLP Research Checklist](https://aclrollingreview.org/responsibleNLPresearch)

## 14. LLM / AI Assistant Usage Policy

### Core Rule

Generative AI tools (ChatGPT, Claude, Copilot, etc.) **do not qualify for authorship**. All use must be disclosed.

### Disclosure Requirements

1. Complete the **E1 section** of the Responsible NLP Checklist
2. Describe usage in the **Acknowledgments** section (camera-ready only; during review, note in the checklist)

### Acceptable Uses

| Use | Disclosure Required? |
|---|---|
| Grammar/spell checking | Optional |
| Language polishing (not new content) | Yes |
| Predictive keyboard suggestions | Optional |
| Literature search assistance | Yes |
| Low-novelty text generation (with citation verification) | Yes |
| Ideation that authors then develop independently | Yes |
| Code generation assistance | Yes |

### Prohibited

- Using AI-generated content without disclosure
- Prompt injections or text designed to manipulate machine readers -- results in **immediate desk rejection**
- Claiming AI-generated text as original work

> **Difference from NeurIPS/ICML:** NeurIPS has a similar policy but less detailed. ACL explicitly lists acceptable use categories and requires disclosure in both the checklist and acknowledgments.

**Source:** [ARR CFP](https://aclrollingreview.org/cfp)

## 15. Supplementary Materials

### Types

1. **Appendices** -- included in the same PDF as the main paper, after references
2. **Software** -- single `.tgz` or `.zip` archive
3. **Data** -- separate single `.tgz` or `.zip` archive

### Rules

- Do **not** count toward page limits
- Reviewers are **not required** to examine supplementary materials
- Must be **fully anonymized** (no author-identifying information)
- Appendices must follow **double-column format** (exceptions allowed for math-heavy content)
  - Starting **July 2025**, appendices not in double-column format face desk rejection
- Software/data must use anonymous hosting (e.g., Anonymous GitHub)
- **Prohibited:** Google Drive, Dropbox, or any cloud service that can track identity
- The paper must be self-contained without the supplements

### What Belongs in Appendices vs. Supplements

- Extended proofs, additional tables/figures, annotation guidelines --> Appendices (in PDF)
- Source code, trained models --> Software archive
- Datasets, raw results --> Data archive

**Source:** [ARR CFP](https://aclrollingreview.org/cfp), [Author Checklist](https://aclrollingreview.org/authorchecklist)

## 16. Author Response Period

### How It Works

1. After reviews are written (~4 weeks into the cycle), authors receive all reviews.
2. Authors have a **7-day window** to write a response.
3. The response is **text-only** -- no external links, no PDF uploads.
4. After the response, reviewers discuss and may update their reviews.
5. The AC then writes a meta-review.

### ACL 2025 Specific

- Author response: **March 27 -- April 3, 2025** (for the February 2025 ARR cycle)

### Strategy Notes

- Address factual errors in reviews directly
- Commit to specific changes ("We will add experiment X") -- reviewers appreciate concrete plans
- Keep it concise; reviewers may not read long responses
- **Violation:** Including external links in the response text results in desk rejection

**Source:** [ARR Dates](https://aclrollingreview.org/dates), [ARR Authors](https://aclrollingreview.org/authors)

## 17. Ethics Review Process

### How It Works

1. Reviewers can **flag** papers for ethics concerns during regular review.
2. Flagged papers are reviewed by the **Ethics Advisory Committee** in parallel with standard review.
3. Ethics reviewers do not share feedback with standard reviewers.
4. ARR reserves the right to reject papers on ethical grounds even if technically sound.

### What Triggers Ethics Flags

- Research involving human subjects without IRB documentation
- Potential for dual-use harm (surveillance, disinformation, deception)
- Privacy violations or inadequate anonymization of research subjects
- Insufficient discussion of societal impact
- Environmental concerns not addressed

### Optional Ethics Section

- Title: "Ethical Considerations"
- Place after conclusion (alongside Limitations), before references
- Does **not** count toward page limits
- Recommended but not mandatory (unlike Limitations)

**Source:** [ARR CFP](https://aclrollingreview.org/cfp), [ARR How It Works](https://aclrollingreview.org/reviewing)

## 18. Conflict of Interest Rules

### OpenReview-Based COI Management

- Authors must maintain **up-to-date OpenReview profiles** with correct:
  - Current and past affiliations (with dates)
  - Semantic Scholar, DBLP, and ACL Anthology links
  - Listed publications reflecting current research interests
- The system uses profile data for automated COI detection and reviewer assignment.

### What Constitutes a COI

ACL follows the ACM Code of Ethics (adopted by ACL). Standard COIs include:
- Co-authors within the last 3 years
- Advisor/advisee relationships
- Same institution or research group
- Close personal relationships
- Financial relationships

### Author Responsibilities

- Ensure all co-authors have complete, accurate OpenReview profiles
- Report COIs that automated systems may miss via the submission form
- Do not attempt to identify or contact reviewers

**Source:** [ARR CFP](https://aclrollingreview.org/cfp), [ACL Code of Ethics](https://www.aclweb.org/portal/content/acl-code-ethics)

## 19. Reviewer Obligation for Authors

> **Major policy change (May 2025):** All submitting authors must sign up as reviewers.

### Requirements

- **All authors** must register as reviewers within **48 hours** of the submission deadline.
- Reviewer assignments are based on qualifications and publication record.
- Teams not meeting this requirement face potential **desk rejection**.

### Qualification Standards

At least:
- 2 short or long papers in main ACL events or Findings
- Plus 1 more paper in the ACL Anthology or a major ML/AI venue

### Exceptions

- Authors new to the field (first submission)
- Authors serving in other ARR capacities (AC, SAC, etc.)

### Consequences of Non-Compliance

- Failure to complete assigned reviews by deadlines may make authors **ineligible to submit in the following cycle**.
- This policy mirrors CVPR's reviewer requirement.

**Source:** [ARR CFP](https://aclrollingreview.org/cfp), [ARR Authors](https://aclrollingreview.org/authors)

## 20. Desk-Rejection Triggers

Papers can be desk-rejected **at any point during the review cycle** for:

### Formatting
- [ ] Exceeding page limits (8 long / 4 short)
- [ ] Wrong paper size (must be A4)
- [ ] Wrong margins, font sizes, or column layout
- [ ] Modified style files or templates from other conferences
- [ ] Missing line numbers in submission version (use `[review]` option)
- [ ] Appendices not in double-column format (enforced starting July 2025)

### Anonymization
- [ ] Author names or identifying information visible
- [ ] Non-anonymous repository links
- [ ] Dropbox, Google Drive, or tracking-capable file services
- [ ] Author identity revealed through self-citations

### Content
- [ ] Missing Limitations section
- [ ] Incorrect, incomplete, or misleading Responsible NLP Checklist responses
- [ ] Paper out of scope for computational linguistics / NLP
- [ ] Prompt injections or text manipulating machine readers

### Policy
- [ ] Dual submission (under review elsewhere)
- [ ] Identical resubmission without revision
- [ ] Violating binding non-preprint commitment
- [ ] Unmarked author list changes in resubmissions
- [ ] Self-plagiarism
- [ ] External links in author response text

### Administrative
- [ ] Author response containing external links (text-only required)
- [ ] No presenting author registered by early registration deadline (post-acceptance)
- [ ] Failure to meet reviewer obligations (starting May 2025)

**Source:** [ARR CFP](https://aclrollingreview.org/cfp)

## 22. Resubmission Policy

If your paper is reviewed and you want to improve it, you may resubmit to a later ARR cycle:

### Requirements

- Provide a **link to the previous submission**
- Include a **summary of changes** addressing prior reviews
- Justify any author list changes
- Cannot resubmit an **identical paper** -- substantive revisions required
- Must address relevant feedback or justify deviation

### Special Rules

- Papers scoring **1/5 on the meta-review** require wholesale revision if resubmitted within 6 months
- Resubmissions typically return to **original reviewers** unless authors request new reviewers with justification

### Withdrawal and Resubmission

- Withdrawal is allowed anytime during review
- However, withdrawing **more than 48 hours after the deadline** prevents resubmission until the **second subsequent cycle** (i.e., you skip one cycle)
- Papers that received at least one review cannot be withdrawn for publication elsewhere without following resubmission policy

**Source:** [ARR CFP](https://aclrollingreview.org/cfp)

## 23. Differences from NeurIPS/ICML

| Aspect | ACL (ARR) | NeurIPS / ICML |
|---|---|---|
| **Review system** | Rolling review + commitment | Single deadline |
| **Paper size** | **A4** | US Letter (8.5" x 11") |
| **Page limit (long)** | 8 pages | 9 pages (NeurIPS), 8+1 (ICML) |
| **Limitations section** | **Mandatory** (desk rejection) | Not mandatory (NeurIPS has broader impact) |
| **Anonymity period** | None | 1 month before -- after decisions |
| **Preprint policy** | Permissive (incentives for anonymous) | More restrictive |
| **Reviewer obligation** | Authors must review | No such requirement |
| **Findings track** | Yes (second-tier acceptance) | No equivalent |
| **Resubmission** | Supported within ARR | Fresh submission each time |
| **Style file** | `acl.sty` (natbib) | `neurips_2025.sty` / `icml2025.sty` |
| **Supplementary format** | Appendices in same PDF | Often separate PDF |
| **Author response** | Text-only, no links | PDF allowed at NeurIPS |
| **Responsible checklist** | Detailed NLP-specific checklist | NeurIPS checklist (different questions) |
| **Ethics review** | Parallel ethics committee | Similar but different structure |
| **Commitment step** | Required (separate from review) | N/A |

## 24. Pre-Submission Checklist

Use this checklist before submitting to ARR:

### Formatting
- [ ] Paper uses official `acl.sty` with `[review]` option
- [ ] Paper is A4 size (verify with `pdfinfo`: 595.276 x 841.89 pts)
- [ ] All fonts embedded (verify with `pdffonts`)
- [ ] Within page limits: 8 pages (long) or 4 pages (short), excluding limitations/ethics/references
- [ ] Line numbers visible (enabled by `[review]` option)
- [ ] Two-column format throughout (including appendices)
- [ ] Correct font sizes for all elements (body 11pt, abstract 10pt, etc.)
- [ ] Abstract under 200 words
- [ ] Figures in vector format where possible; grayscale-readable
- [ ] Run `aclpubcheck` and resolve all errors

### Content
- [ ] **Limitations section** present after conclusion, before references
- [ ] No page break between conclusion and limitations
- [ ] Optional "Ethical Considerations" section present if applicable
- [ ] References, limitations, ethics, appendices all in one PDF

### Anonymization
- [ ] No author names or affiliations anywhere in the PDF
- [ ] No acknowledgments section
- [ ] Self-references in third person
- [ ] No links to identifiable repositories
- [ ] Supplementary materials anonymized
- [ ] No identifying info in PDF metadata
- [ ] Preprint status field set correctly

### Bibliography
- [ ] DOIs included where available
- [ ] Full author names used (not initials)
- [ ] Alphabetical order by first author
- [ ] Using `acl_natbib.bst`
- [ ] Preprints cited as refereed versions where published

### Checklist & Policies
- [ ] Responsible NLP Research Checklist completed accurately in OpenReview
- [ ] AI/LLM usage disclosed in checklist section E1
- [ ] Paper not under review at any other venue
- [ ] If resubmission: link to previous version + change summary provided
- [ ] If resubmission: author list changes documented in submission form

### Administrative
- [ ] All co-authors listed in correct order (cannot change post-submission)
- [ ] All co-authors have complete OpenReview profiles with correct affiliations
- [ ] All co-authors prepared to register as reviewers within 48 hours
- [ ] Submitted at least 1 hour before deadline
- [ ] Selected correct paper type (long/short) and track

### Post-Submission (within 48 hours)
- [ ] All authors registered as reviewers on OpenReview
- [ ] OpenReview profiles updated with Semantic Scholar, DBLP, ACL Anthology links
