## Work Completed

### 1. Formal Proofs for Propositions (notes/07-proposition-proofs.md)

Developed rigorous proofs and detailed proof sketches for all 5 core propositions:

**Proposition 1 (Sensitivity Gap Existence)**
- Proved that functions with boolean sensitivity $s(f) = \Omega(n)$ exist outside transformer capability
- Connected to Linial-Mansour-Nisan results on AC⁰ limitations
- Incorporated Hahn & Rofin learnability barriers

**Proposition 2 (Depth Gap Existence)**
- Proved conditional existence of NC¹ problems unsolvable by TC⁰ transformers
- Showed O(log n) CoT steps suffice to close depth gaps
- Formalized as conditional on TC⁰ ≠ NC¹ (standard assumption)

**Proposition 3 (Serial Gap Characterization)**
- Tight bound: Ω(k) CoT steps necessary and O(k) sufficient for k-fold composition
- Information-theoretic argument for lower bound
- Constructive upper bound via CoT algorithm

**Proposition 4 (CoT Monotonicity)**
- Proved accuracy monotonically non-decreasing with budget up to optimal
- Formalized "overthinking" phenomenon as accuracy degradation beyond optimal budget
- Specified empirical test for validation

**Proposition 5 (Intractability Invariance)**
- Proved poly CoT cannot systematically solve NP-hard instances at phase transition
- Connected to P ≠ NP assumption
- Showed accuracy bounded away from 1 for large n

**Additional Corollaries**:
- Corollary 1: Architectural gaps are scale-invariant (proved)
- Corollary 2: Sensitivity and algorithmic gaps have mixed scale dependence (characterized)

**Impact**: Strengthens theoretical contribution. All propositions now have formal justification grounded in complexity theory.

### 2. Empirical Analysis Plan (notes/08-empirical-analysis-plan.md)

Comprehensive specification of all analyses to be performed once evaluation data available:

**Data Collection Protocol**:
- 6,900 benchmark instances across 9 tasks
- 12 model configurations (Claude, GPT, Llama, Mistral, Qwen)
- 4 evaluation conditions (Direct, Short CoT, Budget CoT, Tool)
- **Total evaluations**: 331,200

**Six Primary Analyses**:
1. **Gap Type Validation**: Verify each task exhibits predicted gap pattern
   - Statistical test: Spearman correlation (negative degradation)
   - Expected: 9 tasks show systematic degradation

2. **CoT Effectiveness by Gap Type**: Quantify CoT lift per gap type
   - Statistical test: One-way ANOVA with Tukey HSD
   - Expected: Lift(Type 2,3) > Lift(Type 1,4) > Lift(Type 5,6)

3. **CoT Budget Sufficiency**: Test Proposition 4 (monotonicity)
   - Statistical test: Friedman test + Jonckheere-Terpstra trend
   - Expected: Plateau at budget matching theoretical prediction

4. **Scale Dependence**: Identify which gaps narrow with model scale
   - Statistical test: Mixed-effects model with size × gap type interaction
   - Expected: Type 5,6 scale-invariant; Types 1-4 moderate improvement

5. **Tool Augmentation**: Quantify tool vs CoT effectiveness
   - Statistical test: Paired t-test with Cohen's d effect size
   - Expected: Large lift for B6 (algorithmic), minimal for B7 (intractable)

6. **Faithfulness Correlation**: Test if CoT faithful when computationally necessary
   - Statistical test: Chi-square for association
   - Expected: Higher faithfulness for Types 2-3

**Three Secondary Analyses**:
- Per-model characterization (gap score heatmap)
- Error pattern analysis (failure mode classification)
- Latency and cost analysis (efficiency vs accuracy tradeoff)

**Robustness Checks**:
- Prompt sensitivity (3 alternative phrasings)
- Bootstrap confidence intervals (1000 iterations)
- Cross-family consistency (mixed-effects with family random effect)

**Visualization Standards**:
- 4 main figures specified (9-panel accuracy curves, CoT lift box plots, scale scatter, tool bars)
- 2 main tables (summary statistics, per-model matrix)
- Export formats, dimensions, color schemes all specified

**Reproducibility**:
- All analyses automated via scripts
- Version-controlled, deterministic (seeded)
- Complete documentation with dependencies

**Success Criteria**:
- 6 explicit criteria for validating taxonomy
- If all met, taxonomy is empirically validated

**Impact**: Pre-registration of analyses prevents p-hacking. Complete specification enables reproducible execution once data available. Estimated 7 days from data collection to paper-ready results.

## Key Insights from This Session

### Theoretical Clarity

The formal proofs clarified several important points:

1. **Conditional Nature**: Most propositions are conditional on standard complexity assumptions (TC⁰ ≠ NC¹, P ≠ NP). This is appropriate and standard for this type of work.

2. **Tightness**: Proposition 3 provides tight bounds—exactly Θ(k) steps needed for k-fold composition. This precision strengthens the contribution.

3. **Learnability vs Expressiveness**: Propositions 1 and 4 distinguish gaps due to expressiveness (hard ceiling) from gaps due to learnability (optimization barrier). This is a key conceptual distinction.

4. **Scale Invariance**: Corollary 1 proves architectural gaps don't improve with scale—important for resource allocation decisions.

### Analysis Plan Rigor

The comprehensive analysis plan provides:

1. **Clear Hypotheses**: Each analysis has explicit H₀ and H₁
2. **Appropriate Tests**: Statistical methods matched to data type and assumptions
3. **Effect Sizes**: All tests include effect size measures (Cohen's d, η²)
4. **Multiple Comparisons**: Bonferroni correction specified where needed
5. **Pre-Registration**: Analyses specified before seeing data (gold standard)
